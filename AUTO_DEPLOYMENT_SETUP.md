# 🤖 Automated Commit & Deploy Setup

## Overview

Your application now has **3 layers of automation**:

1. **File Watcher** (optional) - Auto-commits when you change files
2. **GitHub Actions** (ready) - Tests and validates on every push
3. **Render Auto-Deploy** (active) - Deploys to production on every push

---

## ⚡ Quick Start (Recommended)

### Option 1: Use File Watcher (Easiest - Recommended for Development)

**Windows:**
```bash
powershell -ExecutionPolicy Bypass -File auto-commit-deploy.ps1
```

**Linux/macOS:**
```bash
bash auto-commit-deploy.sh
```

Leave this running in a terminal. It will:
- ✅ Watch for file changes every 5 seconds
- ✅ Auto-commit when you make changes
- ✅ Auto-push to GitHub
- ✅ Trigger GitHub Actions (tests)
- ✅ Auto-deploy to Render (1-2 minutes later)

---

## 🔄 How It Works

```
You edit a file
    ↓
File watcher detects change
    ↓
Auto-stages files (git add)
    ↓
Auto-commits (git commit)
    ↓
Auto-pushes to GitHub (git push)
    ↓
GitHub Actions runs (tests build)
    ↓
Render webhook receives push
    ↓
✅ Auto-deploys to production
    ↓
✨ Live in 1-2 minutes
```

**What:** Render's automatic GitHub integration
**Trigger:** Successfully passed GitHub Actions

**Status:** ✅ Already active

## ⚙️ Configuration Options

### File Watcher Behavior

Edit `auto-commit-deploy.ps1` or `auto-commit-deploy.sh`:

```powershell
# Change watch interval (default: 5 seconds)
[int]$IntervalSeconds = 5

# Change watched directory
[string]$WatchPath = "d:\Learn\iternary\itinerary-backend"

# Change Git repository path
$GitPath = "d:\Learn\iternary"
```

### GitHub Actions Configuration

Edit `.github/workflows/deploy.yml`:
- Change test timeout
- Add additional build steps
- Change Go version
- Add environment variables

### Render Webhook Configuration

https://dashboard.render.com → Your Service → Settings → "Deploy Hook" (if using manual triggers)

## 🆘 Troubleshooting

### File Watcher Not Detecting Changes

**Solution 1:** Make sure the file watcher script is running
```bash
# Check running processes
Get-Process | Select-String "powershell"
```

**Solution 2:** Verify watch path is correct
```bash
# In the script, check:
$WatchPath = "d:\Learn\iternary\itinerary-backend"  # Path must exist
$GitPath = "d:\Learn\iternary"                      # Git repo here
```

### GitHub Actions Failing

Check logs: https://github.com/agamrai0123/iternary/actions

Common issues:
- Go version mismatch
- Missing dependencies
- Test failures
- Build errors

### Render Deployment Failing

Check logs: https://dashboard.render.com → Your Service → Logs

Common issues:
- Build command failed
- Port binding error
- Database connection failed
- Environment variables not set
